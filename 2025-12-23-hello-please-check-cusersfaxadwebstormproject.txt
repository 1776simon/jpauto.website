  Photo Overlay - CSS vs. Baked-In Solution

  The Problem with CSS Overlays:
  CSS overlays (like the vehicle info bar at the bottom of photos) only
  exist as a visual layer in the browser. When someone downloads or shares
  the image, the CSS overlay disappears because it's not part of the actual
  image file - it's just rendered on top in the browser.

  To "Bake" Overlays into Images:
  You need server-side image processing that composites the overlay
  text/graphics directly into the image file before it's saved. This cannot
  be done with CSS alone.

  Solution Options:

  1. Backend Image Processing (Recommended)
    - Use your Node.js backend with Sharp library (already used in your
  inventory system)
    - When images are uploaded, automatically composite the overlay onto
  each image
    - Location: inventory-system/server/services/imageProcessor.js already
  has Sharp configured
    - Example code:
  const sharp = require('sharp');

  // Composite overlay onto image
  const overlayBuffer = await generateOverlayImage({
    year: vehicle.year,
    make: vehicle.make,
    model: vehicle.model,
    stockNumber: vehicle.stockNumber
  });

  await sharp(inputImage)
    .composite([{
      input: overlayBuffer,
      gravity: 'south' // position at bottom
    }])
    .toFile(outputPath);
  2. Pre-Processing Before Upload
    - Use tools like ImageMagick, Photoshop Actions, or Python (Pillow) to
  batch-process images before uploading
    - Requires manual step but gives full control over design
  3. Client-Side Canvas Approach (Less ideal)
    - Use HTML5 Canvas to generate composite images on-demand
    - Works but adds complexity and processing time

  My Recommendation:
  Extend your existing imageProcessor.js service in the backend to add
  overlay text when processing vehicle photos. This way, every uploaded
  image will have the overlay permanently burned into it, and it will appear
   in downloads, social media shares, etc.

  Would you like me to implement this server-side image overlay feature in
  your backend?
